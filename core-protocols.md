# Core Protocols — integration reference

> **The Core Protocols** by **Jim and Michele McCarthy** — *Software For Your Head*.
> License: **GPL v3+** — <https://www.gnu.org/licenses/>
> Source (full text and mechanics): <https://liveingreatness.com/core-protocols/>

This file lists the protocols by name with a one-line "when to invoke" written in our own words. **The protocol mechanics (wording, response rules, hand signals) live at the source URL — we do not reproduce them here.** That is what keeps this skill MIT-licensed (reference, not derivation). Read the source before using a protocol for the first time.

## How this layers on Cardon's Delegated Processes

| Layer | Practice | What it controls |
|-------|----------|------------------|
| **Structure** | Cardon — Delegated Processes | Who holds which rotating role this session. |
| **Behavior** | McCarthy — Core Protocols | How every person, in any role, shows up and interacts. |

The two practices are orthogonal and stack cleanly. Cardon does not prescribe how attendees should behave inside the meeting beyond the role descriptions; the Core Protocols fill exactly that gap.

## In-meeting integration points

| Cardon moment | Core Protocol that plugs in | The moment in plain language |
|---------------|-----------------------------|-------------------------------|
| Meeting open (Facilitator cues it) | **Check In** | Quick round so everyone arrives present rather than half-in-an-email. |
| Anytime, any attendee | **Pass / Unpass** | Decline a turn without owing an explanation; rejoin later if you change your mind. |
| Anytime, any attendee | **Ask For Help** | When you need input from a specific teammate, ask them by name. |
| Anytime, any attendee | **Protocol Check** | When you observe a behavior that breaks an agreed protocol. Distributes enforcement away from the Facilitator. |
| Anytime, any attendee | **Intention Check** | When the purpose behind someone's contribution is unclear and you need it surfaced. |
| Decision Driver, when consensus stalls on a binary proposal | **Decider** | Replaces ad-hoc "are we close to a decision?" with a structured group decision protocol. |
| Decision Driver, after a No vote in Decider | **Resolution** | The follow-up protocol that converges the disagreement. |
| Process Coach, final 10 minutes | **Perfection Game** | Optional, more structured than free-form feed-forward. One per attendee. |
| Anyone who genuinely cannot add value | **Check Out** | Leave (or temporarily disengage) explicitly rather than disengage silently. |

## Other protocols (mostly out of scope for a single in-meeting use)

These protocols are part of the same body of work but are typically used outside a single meeting (team formation, 1:1 coaching, individual goal-setting). Mention them by name to attendees if relevant, link to the source URL, do not pull them into the meeting agenda by default:

- **Personal Alignment**, **Personal Alignment Express**
- **Investigate**
- **Personal Commitments form**, **Web of Commitment Express**
- **Meet**, **Listen**

Adopt incrementally. A team brand-new to both Cardon and the Core Protocols should start with **Check In** and **Decider** only — that already changes the texture of meetings significantly.

Every Confluence page the skill creates carries the McCarthy attribution and source URL alongside the Cardon attribution — even if no Core Protocol is invoked in that session. Both attributions sit in the same panel at the top of the page. If you reference a specific protocol in the page body, link to the source URL on first mention.

## Why we do not ship the source PDF

The Core Protocols are GPL v3+. Distributing a copy (PDF or reproduced text) makes this skill a derivation, which would force the whole skill under GPL and require redistributing the McCarthy source. By referencing protocols by name + linking to the source URL, the skill remains MIT-licensed and stays fully GPL-compliant by simply not creating a derivation.
